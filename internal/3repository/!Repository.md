# 💾 repository/ (ทาสผู้ทำงานให้ Database)

## 📌 หน้าที่
รับผิดชอบการ "คุยกับแหล่งข้อมูล (Data Source)" โดยตรงเท่านั้น ไม่ว่าจะเป็น PostgreSQL, MySQL, Redis, หรือระบบ API ภายนอกอื่นๆ

มันรับหน้าที่แปลงข้อมูลจากฐานข้อมูลของจริง ให้ออกมาเป็น `Struct` ที่กำหนดไว้ในฝั่ง `domain`

## ❌ ข้อห้าม
- **ห้ามใส่ Business Logic** : เช่น ถ้าผู้ใช้แบนอยู่ห้ามดึงข้อมูล (แบบนี้ห้ามทำในนี้ ให้ไปทำใน `usecase` ที่นี่มีหน้าที่แค่ โดนสั่งให้ดึงก็ดึง โดนสั่งให้เซฟก็เซฟ)
- **อย่าให้ข้อมูลรั่วไหล** : ห้ามหลุดคำสั่ง SQL ออกไปนอกโฟลเดอร์นี้ ฝั่งอื่นของแอปต้องไม่ล่วงรู้ว่าเราใช้ Postgres หรือ MySQL

## 💡 ตัวอย่าง
ไฟล์ `note_repo.go` จะเป็นการเขียนโค้ด SQL:
`SELECT * FROM notes WHERE id = 1` 
จากนั้นก็ยัดค่าใส่ตัวแปร `domain.Note` แล้วก็ส่งคืนออกไป แค่นั้นเลย
