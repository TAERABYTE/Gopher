# 👑 domain/ (หัวใจของ Clean Architecture)

## 📌 หน้าที่
โฟลเดอร์นี้คือ **หัวใจหลักและสิ่งสำคัญที่สุดของโปรเจกต์**
ถ้าเดินเข้ามาดูโฟลเดอร์นี้ คุณต้องรู้ทันทีว่าโปรเจกต์นี้ทำอะไร (เช่น อ๋อ มี User นะ มี Note นะ มี Auth นะ)

โฟลเดอร์นี้ใช้เก็บ:
1. **Struct (Data Object)** : หน้าตาโครงสร้างข้อมูล เช่น ข้อมูล User หรือ Note ประกอบด้วยอะไรบ้าง
2. **Interface (สัญญาหน้าฝน)** : การประกาศฟังก์ชันว่า "ในโปรเจกต์ฉันต้องมีเมธอดแบบนี้นะ" (เช่น `GetByID`, `Create`)
3. **Custom Errors** : ข้อผิดพลาดต่างๆ (เช่น `ErrNotFound`)

## ❌ ข้อห้าม (สำคัญมาก!)
- **ห้ามพึ่งพา (Import) โฟลเดอร์อื่นเข้ามาในนี้เด็ดขาด** (ยกเว้น Standard Package ของ Go เช่น `context`, `time`)
- โฟลเดอร์ `domain/` ไปรู้อะไรของฝั่ง Database (`sql`), HTTP (`net/http`) ไม่ได้ทั้งสิ้น มันเป็นอิสระจากทุกกรอบ

## 💡 ตัวอย่าง
ไฟล์ `user.go` ใน `domain/` ไม่ได้เขียนโค้ดต่อ Postgres แต่เขียนแค่ **สัญญา (Interface)** ว่า:
"ใครก็ตามที่อยากเป็น `UserRepository` ให้ฉัน เขาต้องมีฟังก์ชัน `GetByUsername` นะ ส่วนเขาจะไปดึงมาจาก Postgres, MySQL, หรือ Text File ก็เรื่องของเค้า ฉันไม่สน"
